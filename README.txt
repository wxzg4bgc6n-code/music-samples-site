NOIRWAVE — загрузка Sample Packs и треков в Backblaze B2

ЧТО ИЗМЕНИЛОСЬ

- все демонстрационные паки, треки, превью и выдуманные счётчики удалены;
- пустой сайт показывает «Пока ничего не опубликовано»;
- owner и admin загружают настоящие файлы через админку;
- обложки, аудио и архивы хранятся в закрытом Backblaze B2;
- карточки, прослушивания и скачивания хранятся в Cloudflare D1;
- каталог общий: добавленное другом-администратором увидят все;
- слушать можно без регистрации;
- скачать архив пака или аудиофайл трека можно только после входа через Google;
- под каждой обложкой трека есть плеер: время, прогресс, громкость и скачивания;
- обложки треков можно скрыть кнопкой над каталогом;
- удаление карточки удаляет связанные файлы из B2;
- существующие пользователи и роли не удаляются.

ФАЙЛЫ

Сайт для GitHub / Cloudflare Pages:
- index.html
- style.css
- script.js
- config.js
- папка admin целиком

Код Cloudflare Worker:
- noirwave-api-worker-b2.js


ШАГ 1. ОБНОВИТЬ WORKER

1. Откройте Cloudflare.
2. Workers & Pages → noirwave-api.
3. Откройте Edit code.
4. Полностью замените текущий код содержимым файла:
   noirwave-api-worker-b2.js
5. Нажмите Deploy.

Worker автоматически создаст в уже подключённой D1 новые таблицы для материалов.
Таблицы пользователей и ролей не очищаются.


ШАГ 2. ПРОВЕРИТЬ ПЕРЕМЕННЫЕ WORKER

Cloudflare → noirwave-api → Settings → Variables and Secrets.

Должны существовать:

GOOGLE_CLIENT_ID
SITE_ORIGIN
OWNER_EMAIL
B2_KEY_ID
B2_APPLICATION_KEY
B2_BUCKET_NAME
B2_ENDPOINT

Значения для хранилища:

B2_BUCKET_NAME = noirwave-files
B2_ENDPOINT = https://s3.us-east-005.backblazeb2.com

B2_APPLICATION_KEY должен быть типом Secret.
B2_KEY_ID можно оставить Plaintext.

В разделе Bindings должна остаться D1-привязка:

Variable name: DB
Database: та же база noirwave, в которой уже находятся пользователи.


ШАГ 3. ОБНОВИТЬ САЙТ

В GitHub замените:

- index.html
- style.css
- script.js
- config.js
- всю папку admin

Файл Worker загружать в GitHub необязательно: он нужен для редактора Cloudflare Worker.

Дождитесь нового деплоя Cloudflare Pages.
Откройте https://music-samples-site.pages.dev и сделайте Ctrl+F5.


ШАГ 4. ПЕРВАЯ ПРОВЕРКА B2

1. Войдите на сайт как owner или admin.
2. Откройте Админка.
3. Админка автоматически настроит в закрытом bucket безопасное правило загрузки
   только с адреса SITE_ORIGIN.
4. На главной странице админки должно появиться:
   «Подключено: noirwave-files».

Если статус красный:

1. Откройте Админка → Настройки.
2. Нажмите «Проверить B2 и настроить загрузку».
3. Проверьте четыре переменные B2 в Cloudflare.


ШАГ 5. ЗАГРУЗИТЬ ПЕРВЫЙ МАТЕРИАЛ

Sample Pack:

1. Админка → Sample Packs → Добавить пак.
2. Заполните название.
3. Выберите ZIP, RAR или 7Z — архив обязателен.
4. Обложка и аудиопревью необязательны.
5. Нажмите «Опубликовать» и не закрывайте окно до окончания загрузки.

Трек:

1. Админка → Треки → Добавить трек.
2. Заполните название.
3. Выберите аудиофайл — он обязателен.
4. Обложка необязательна.
5. Нажмите «Опубликовать».

Один файл может быть размером до 5 ГБ.
Файлы отправляются напрямую из браузера в закрытый B2 по временной ссылке.
Секрет B2 никогда не передаётся в браузер.


ПРОВЕРКА

1. Материал появился в админке.
2. Откройте основной сайт и обновите страницу.
3. Карточка появилась в каталоге.
4. Аудио запускается.
5. Число прослушиваний увеличивается после начала воспроизведения.
6. Без входа скачивание предлагает войти.
7. После входа архив скачивается.
8. Число скачиваний увеличивается.
9. Второй администратор видит тот же материал.


ВАЖНО О БЕЗОПАСНОСТИ

Application Key Backblaze ранее был показан на скриншоте. После успешной проверки
рекомендуется создать новый ключ noirwave-worker с доступом Read and Write только
к bucket noirwave-files, заменить B2_APPLICATION_KEY и B2_KEY_ID в Cloudflare,
а старый ключ удалить в Backblaze.

Client ID в config.js публичный — это нормально.
Application Key и другие секреты нельзя добавлять в GitHub или файлы сайта.
