Что уже подключено:
- просмотр сайта без регистрации;
- первый вход через Google автоматически создаёт пользователя в D1;
- аккаунт с почтой из OWNER_EMAIL получает роль owner;
- остальные новые аккаунты получают роль user;
- кнопка «Админка» появляется только у owner/admin;
- страница /admin/ проверяет сессию через Worker;
- скачивание требует входа в аккаунт.

Что пока остаётся демонстрационным:
- паки и треки из админки сохраняются только в localStorage браузера;
- ZIP, MP3 и обложки ещё не загружаются в R2;
- общие счётчики скачиваний/прослушиваний ещё не подключены к API;
- серверный список пользователей и назначение роли admin будут следующим этапом;
- регистрация по почте с кодом отложена до появления собственного домена.

Как обновить сайт:
1. Распаковать архив.
2. Загрузить в GitHub все файлы и папку admin, сохранив структуру.
3. Дождаться нового деплоя Cloudflare Pages.
4. Открыть https://music-samples-site.pages.dev и выполнить жёсткое обновление Ctrl+F5.
5. Нажать «Войти» → «Продолжить через Google».
6. Пока OAuth находится в режиме Testing, Google-аккаунт должен быть добавлен в Audience → Test users.

Проверка:
- после первого входа /api/db-test должен показать users: 1;
- у OWNER_EMAIL должна появиться роль owner;
- ссылка «Админка» должна появиться в меню;
- /admin/ должна открываться только для owner/admin.

Безопасность:
- Client ID можно хранить в config.js: он публичный;
- Client secret нельзя добавлять на сайт;
- сессия сейчас хранится как Bearer-токен в localStorage;
- перед публичным запуском позже лучше перейти на HttpOnly Secure cookie.
